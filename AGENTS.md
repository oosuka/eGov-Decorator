# AGENTS.md

## 言語

- このリポジトリでの既定言語は日本語です。
- 通常の回答、作業報告、提案、コミットメッセージ案は日本語を優先してください。
- ユーザーが別言語を明示した場合のみ、その言語に切り替えてください。

## `/review` の出力ルール

- `/review` を求められた場合、レビュー結果は日本語で出力してください。
- まず重大度順に、バグ、回帰リスク、仕様漏れ、テスト不足を指摘してください。
- 指摘には可能な限りファイルパスと行番号を含めてください。
- 問題が見つからなかった場合も、その旨を日本語で明記し、残るリスクや未確認事項があれば添えてください。
- `/review` の後、指摘事項が局所的で、仕様が明確で、低リスクに修正できる場合は、自動で修正まで進めてください。
- `/review` の後、仕様判断が必要なもの、影響範囲が広いもの、後方互換やUI挙動に影響しうるものは、修正前に確認してください。
- 自動修正した場合は、レビュー結果と修正内容を分けて日本語で報告してください。

## リポジトリ概要

- Chromium 系ブラウザ向けの拡張機能です。
- 対象は e-Gov 法令ページで、全角括弧 `（...）` をハイライトします。
- 主な実装箇所:
  - `src/content.js`: ページ上のハイライト処理本体
  - `src/background.js`: バッジ更新、ショートカット、タブ同期
  - `src/options.js` / `src/options.css`: 色設定 UI
  - `tests/*.test.js`: `node:test` ベースの単体テスト

## 作業ルール

- 作業開始前に `git status` を確認し、既存の未コミット変更を把握してください。
- 自分の作業と無関係な差分は巻き込まず、既存変更を勝手に巻き戻さないでください。
- 変更は既存の実装方針に合わせ、不要な構造変更は避けてください。
- コミットメッセージは日本語1行で記述してください。
- コミットはできるだけ1目的1コミットにしてください。
- 互換性のために保持している `chrome.storage.local` のキー
  - `highlightLevel`
  - `decoratorEnabled`
  を壊さないでください。
- ブラウザ拡張なので、対象 URL 判定、タブイベント、`storage` 同期の挙動を壊しやすい変更には注意してください。
- `manifest.json` の権限、background、content_scripts、action の整合性を壊さないでください。
- DOM 変更を伴う修正では、ノードまたぎ括弧処理と既存ハイライト解除の挙動を確認してください。

## 確認コマンド

- テスト: `npm test`
- 総合チェック: `npm run check`
- lint: `npm run lint`
- 整形: `npm run format`

## 変更後の確認

- コード変更時は、影響範囲に応じて少なくとも関連テストを実行してください。
- JavaScript のロジック変更時は `npm test` を基本とし、広めの変更では `npm run check` の実行も検討してください。
- `husky` によりコミット時に `biome check` が走るため、整形崩れを残さないでください。
- 拡張の挙動に関わる修正では、必要なら README の説明との不整合も確認してください。
- ユーザー向けの挙動、設定項目、ショートカット、インストール手順を変えた場合は `README.md` も更新してください。
